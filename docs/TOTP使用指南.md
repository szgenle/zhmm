## 一、什么是 TOTP？它能帮你做什么？

TOTP（Time-based One-Time Password，基于时间的一次性密码）是业界最主流的 2FA（二次验证）方案，**每 30 秒**自动变化一个 6 位数字码，作为你账号密码之外的**第二道防线**：

- 哪怕密码泄露，攻击者没有你的手机/设备里的 TOTP Secret，也无法登录
- 兼容 GitHub / 阿里云 / 腾讯云 / 微软账号 / AWS 等几乎所有主流服务

`zhmm` 把 TOTP 能力**内置到密码管理器本体**中，好处：
- 密码与动态码**一处管理**，不用再装 Google Authenticator
- Secret 与密码一起受 **argon2id + SM4-CBC + HMAC-SM3** 保护
- 支持**国密 SM3-TOTP 扩展**（RFC 6238 私有扩展算法名 `SM3`），面向国密合规场景

## 二、前置：获取一个 TOTP Secret

TOTP 的秘钥**不是 zhmm 生成的**，而是**目标网站/服务**在你开启 2FA 时给你的。常见两种形式：

### 形式 A：Base32 明文字符串

网站一般会写「无法扫描二维码？点此手动输入」之类的入口，给你一串字母数字，例如：

```
JBSWY3DPEHPK3PXP
```

或带空格分组（都支持）：

```
JBSW Y3DP EHPK 3PXP
```

### 形式 B：otpauth:// URI

二维码背后的文本内容就是这种标准 URI：

```
otpauth://totp/GitHub:you@example.com?secret=JBSWY3DPEHPK3PXP&issuer=GitHub&algorithm=SHA1&digits=6&period=30
```

如果你能用手机「扫一扫」读出二维码文本，**直接复制整条 URI** 给 zhmm，它会自动解析各字段。

## 三、GUI 使用流程（推荐）

### 1. 打开「添加密码」或「编辑密码」对话框

在密码主界面点 **「添加」** 按钮，或 **双击** 已有记录进入编辑。

### 2. 展开「动态码 / TOTP」折叠区

对话框中有一个独立的 TOTP 分组，包含四个字段：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| Secret | Base32 秘钥，或整条 `otpauth://` URI | 空 |
| 算法 | `SHA1` / `SHA256` / `SHA512` / `SM3` | SHA1 |
| 位数 | 动态码位数 | 6 |
| 周期 | 每多少秒刷新一次 | 30 |

### 3. 录入 Secret（二选一）

**方式 A：粘贴 Base32**
- 把 `JBSWY3DPEHPK3PXP` 粘到 Secret 输入框
- 空格、大小写、缺失的 `=` padding **都会自动容错**
- 其它字段保持默认即可

**方式 B：粘贴 otpauth:// URI（省心）**
- 把整条 `otpauth://totp/...` 粘到 Secret 输入框
- **algo / digits / period 自动回填**，不需要手工对照

### 4. 实时验证

对话框底部会**实时显示当前动态码**，例如 `482913`。

- 若一直显示 `——` 或 `-- -- --`：说明 Secret 不是合法 Base32，请核对
- 若显示正常但与目标网站对不上：请先看第七章「常见问题排查」

确认无误后点 **保存**。

### 5. 日常使用：列表里直接复制

保存后回到主界面，密码表格会多出一列 **「动态码」**（约第 6 列）：

```
482913  28s
```

- **每秒自动刷新**（剩余秒数 28 → 27 → 26 …）
- **单击该单元格 = 复制动态码到剪贴板**
- **10 秒后自动清空剪贴板**，防止遗忘在公用电脑上
- 未设置 TOTP 的行该列为空，不会干扰

## 四、CLI 使用流程

CLI 当前**仅支持查询**（新增 / 编辑 TOTP 请用 GUI）。

### 1. 找到记录 ID

```bash
zhmm-cli -i ~/zhmm.zmb --account you@example.com -l
```

输出的第一列就是记录 ID。

### 2. 查询当前动态码

```bash
zhmm-cli -i ~/zhmm.zmb --account you@example.com --totp 12
```

输出示例：

```
482913  (剩余 28s, algo=SHA1)
```

可以配合系统剪贴板命令，例如 macOS：

```bash
zhmm-cli -i ~/zhmm.zmb --account you@example.com --totp 12 | awk '{print $1}' | pbcopy
```

## 五、关于国密 SM3-TOTP 扩展

`zhmm` 在 RFC 6238 标准之外，额外提供一个 `algorithm=SM3` 选项：

- **它是 zhmm 的私有扩展**，对应 HMAC-SM3
- Google Authenticator / Microsoft Authenticator / 1Password 等**第三方认证器不识别 SM3**
- 适合以下场景：
  - 面向国密合规审计，不希望密码栈里出现 SHA-family
  - 自研内部系统对接 zhmm 作为唯一认证器

**日常面向互联网站点，请保持默认的 `SHA1`**，不要手贱改成 SM3，否则会与对方服务器算出的码不一致。

## 六、备份与导出的安全策略

| 场景 | `totp_secret` 是否保留 | 说明 |
|------|----------------------|------|
| `.zmb` 加密备份 | ✅ **保留** | argon2id + SM4-CBC + HMAC-SM3 加密，迁移首选 |
| Excel (.xlsx) 导出 | ❌ **刻意抹除** | 仅导出 `totp_algo / totp_digits / totp_period` 三列元信息 |
| Excel 导入 | 忽略 TOTP 三列 | 导入后需重新在 GUI 里粘贴 Secret |

**为什么这样设计？**

Excel 是明文文件，随手发邮件、扔云盘、截图分享都可能泄露。Secret 一旦明文落盘，2FA 就形同虚设。因此：

- **跨机迁移 / 备份请用 `.zmb`**
- **Excel 仅用于密码本身的备份与人类可读审计**

## 七、常见问题排查

### Q1：对话框里预览码一直是 `——`
- Secret 不是合法 Base32：核对字符集（只能是 `A–Z`、`2–7`）
- 粘贴了错误格式的内容（比如把密码误当 Secret）

### Q2：码生成出来了，但网站说不对
按概率排序：

1. **系统时间不准** — TOTP 强依赖时钟。开启系统自动对时（macOS：设置 → 日期与时间 → 自动设置；Windows：「日期和时间」设置 → 立即同步）
2. **算法选错了** — 绝大多数网站用 `SHA1`；若你手动改成了 `SHA256 / SHA512 / SM3`，请改回 SHA1
3. **位数/周期选错了** — 99% 的站点是 6 位 / 30 秒
4. **Secret 抄错了** — 重新粘贴一遍，或扫码重获 `otpauth://` URI

### Q3：主表格里没有「动态码」列
- 检查是否是旧版本数据：打开一条记录点保存，数据库会自动升级字段
- 表头可被拖动/隐藏，右键表头可重置列显示

### Q4：想把 TOTP 同时录到 Google Authenticator 里做异地双份冗余
完全可以：**同一个 Secret 可以被任意多个认证器持有**，它们算出来的码**完全一致**。前提是两边 algo / digits / period 都要对齐（默认 SHA1 / 6 / 30 就天然对齐）。

### Q5：粘贴 otpauth URI 没有自动回填
- URI 必须以 `otpauth://totp/` 开头（注意是 `totp` 不是 `hotp`）
- URI 里的 `secret=` 必须是合法 Base32

### Q6：能不能扫二维码？
当前版本**不内置二维码识别**（避免引入 OpenCV/ZXing 等重依赖）。请用手机「扫一扫」把二维码文本复制出来，或让网站提供「手动输入」的 Base32 明文。

## 八、安全使用建议

1. **主密码要强**：TOTP Secret 最终受主密码保护，主密码弱 = 整套 2FA 弱
2. **Secret 一定要手抄/扫码当场录入**，不要截屏不要发聊天
3. **开 2FA 前务必保存网站给的「备用恢复码」**，手机丢失/电脑坏掉时用它重置
4. **跨设备用 `.zmb`**，绝不用 Excel 同步 Secret
5. **公用电脑用 CLI `--totp` 查询完后，清屏或关掉终端**，避免动态码残留在回滚缓冲
6. **国密场景**：内部系统对接时使用 `SM3`，但仍建议保留一条 SHA1 作为应急通道
